# 手动测试 Checklist

每次大重构(尤其是前端/IPC)后,按这个清单跑一遍可以快速发现回归。每一项都标了**预期表现**,看到就打勾,看不到就报缺陷。

启动:`cd desktop && npm run tauri:dev`(开发模式,改代码自动热重载)。
关闭时记下窗口位置/大小供后面"窗口状态恢复"项验证。

---

## 0. 启动 & 首屏

- [ ] **首次启动**:窗口默认大小 1180×800,居中;Artists tab 默认选中,列表为空时显示"暂无"占位
- [ ] **日志面板**:Logs tab 应出现一条 `Desktop GUI ready` info 行
- [ ] **状态栏**:左下角应出现 `状态: Ready / 准备就绪`,中间显示 `0 artists / 0 selected`
- [ ] **窗口状态恢复**:关掉窗口后再启动,**位置 / 大小 / 是否最大化**应与上次关闭时一致
  - 进阶:把窗口拖到第二块显示器→关闭→拔掉那块显示器→启动,窗口应回到主屏中央(而不是消失在屏幕外)

## 1. Artists tab(主列表)

### 列表渲染 / 排序
- [ ] **列头排序**:点 ID / Name / Works / New Works 四个列头,箭头切换 ▲▼,排序方向正确;再点一次切换方向
- [ ] **虚拟滚动**:艺术家数量很多时(>100),滚动流畅,不卡顿;滚动条不抖
- [ ] **行高亮**:勾选一行后整行高亮(checked 样式)

### 搜索栏
- [ ] **关键字搜索**:输入艺术家名/ID 的一部分,列表实时过滤;清空后恢复全部
- [ ] **路径搜索**:输入 save_path 里出现的文件夹片段,匹配正常
- [ ] **× 清空按钮**:输入框右侧出现 ×;点击立即清空并显示全部
- [ ] **搜索后全选只选可见**:输入关键字过滤后,点"全选",**只选中过滤后的艺术家**(不是整个数据库)

### 收藏 & 标签
- [ ] **收藏星标**:点击行最左侧星标→立即点亮(金色);再点取消;重启后状态保留
- [ ] **只看收藏**:点工具栏星标按钮→列表只剩收藏的艺术家;星标列可排序
- [ ] **新建标签**:展开标签栏→点 `+`→输入名字→出现彩色 chip(颜色按名称固定)
- [ ] **拖拽打标签**:勾选多个艺术家→拖到某个标签上→标签计数增加,这些艺术家获得该标签
- [ ] **标签筛选(多选)**:点多个标签→列表显示带任一所选标签的艺术家;再点取消
- [ ] **重命名 / 删除标签**:右键 chip→重命名或删除;删除会从所有艺术家移除该标签
- [ ] **重启保留**:新建标签后重启,标签栏应直接显示已有标签(不需要再新建才出现)

### 双击行为
- [ ] **双击行**:打开该作者的 Pixiv 主页(用 Settings 里配置的浏览器)
- [ ] **双击 save_path 列**:**用资源管理器打开该文件夹**(不是 Pixiv 页)
- [ ] **右键菜单**:右键单行出现 "修改 ID / 修改保存路径";点击外部关闭菜单

### 工具栏按钮
- [ ] **扫描**:触发 scan(见 §2)
- [ ] **检查更新**:对**已选中**的艺术家拉远端;无选中时对全部
- [ ] **下载更新**:下载已检查到的 new_work_ids
- [ ] **添加艺术家**:Prompt modal 出现三字段(ID / 名字 / 保存路径),最后一个字段右侧有"浏览"按钮可选目录
- [ ] **复制 URL**:剪贴板得到选中艺术家(无选中时全部)的 Pixiv URL,顶部出现 toast `复制了 URL: N`
- [ ] **打开选中**:批量打开 Pixiv 页
- [ ] **删除选中**(红色按钮):无选中时灰色不可点;有选中时弹 ConfirmModal 列出最多 8 个名字 + `... +N`;确认后从数据库移除,toast 显示移除数量

## 2. 扫描 + Scan Preview Modal

- [ ] 点"扫描" → 进度条出现 → 完成后弹 **ScanPreviewModal**(没有变更时只弹日志不弹窗)
- [ ] Modal 顶部有 `选中 / 总数` 计数
- [ ] **默认选中规则**:新艺术家、新作品 ID 默认勾选;名字变更、新增保存路径**默认不勾**(保护手动改过的数据)
- [ ] **分组操作**:每组(新艺术家 / 新作品 / 名字变更 / 新增保存路径)有 "本组全选 / 本组取消" 两按钮
- [ ] **顶部操作**:顶部"全选 / 取消全选"对所有分组生效
- [ ] **逐条勾选**:点 checkbox 切换单条
- [ ] **应用**:点应用,只有勾选项落地;footer 出现 `已应用: N (新增: ..., 名字: ..., 路径: ..., 作品: ...)`
- [ ] **取消**:不写库,关闭 modal
- [ ] 应用后切回 Artists tab,新增/变更已反映

## 3. Unmatched(未识别文件夹)

- [ ] 扫描完成后,如果有未识别文件夹,Unmatched tab 列出它们 + 文件数 + 两个操作按钮
- [ ] **图库根目录不应出现在此列表**(只有子目录)
- [ ] **指派艺术家**(蓝色):弹 PromptModal 输入 ID → 该文件夹挂到该艺术家,该行从列表消失
- [ ] **排除文件夹**(红色):路径加入 settings.exclude_roots,该行从列表消失;切到 Settings → 文件夹 应能看到该路径
- [ ] **关闭再启动**:Unmatched 列表应保留(走 localStorage 缓存)

## 4. Similar(相似图)

### 输入区
- [ ] **扫描根/排除根**:两个 textarea 可以多行编辑
- [ ] **加文件夹按钮**:点击弹出系统选择器,选中后路径追加到对应 textarea
- [ ] **拖拽改高度**:手动拉伸 textarea 高度,关闭应用再启动应记住高度
- [ ] **阈值切换**:Likely / Possible 二选一,segmented 按钮即时切换
- [ ] **Skip Pixiv pages 复选框**:状态会被持久化

### 扫描 & 结果
- [ ] **触发扫描**:进度条经过 "indexing → matching → done" 三段
- [ ] **结果列表**:按组显示,组行可点击展开/折叠;展开状态会被持久化(再启动应保留)
- [ ] **缩略图**:展开后的条目左侧出现缩略图(首次需要等几秒;缓存命中后秒出)
- [ ] **缩略图失败**:对应不存在的图,显示 `!` 占位符,不崩
- [ ] **点缩略图**:弹 ImagePreviewModal 显示大图(1200px);modal 下方有"打开文件夹"按钮
- [ ] **对比模式**:同组多图时,顶部出现「单图 / 并排 / 滑块 / 差异」四个切换;数字键 1-4 可切换
- [ ] **并排**:左右各一张图,下方显示分辨率/大小/修改时间;后端推荐保留的那张有「建议保留」徽标 + 高亮边框
- [ ] **缩放/平移**:滚轮在画面任意处缩放(朝光标方向),放大后可拖动平移;并排时两张图同步缩放/平移;「重置视图」按钮(或按 0 / R)恢复
- [ ] **滑块**:拖动中间竖线像「前后对比」一样擦出差异;拖动滑块不会误触发平移
- [ ] **差异图**:两张分辨率不同的图,差异图按基准图尺寸对齐后再做差分(不再是黑底居中)
- [ ] **方向键 / Esc**:← → 切换上一张/下一张;Esc 关闭;焦点在下拉框时方向键不抢占
- [ ] **双击条目**:用资源管理器选中该文件
- [ ] **关闭再启动**:相似图结果应保留(走 localStorage)

### 清理与隔离
- [ ] **推荐说明**:每组显示推荐规则;顶部按完全重复 / 高度相似 / 可能相似统计组数和预计可释放空间
- [ ] **清理确认**:确认窗口分别列出本次保留和隔离的文件,并显示文件数、总大小与隔离目录
- [ ] **至少保留一张**:不能把同组全部文件选为隔离目标
- [ ] **隔离任务目录**:清理完成后可直接打开该次任务目录;部分失败时显示失败数量与原因
- [ ] **恢复 / 永久删除**:恢复不覆盖已有文件;永久删除仅作用于隔离区并要求二次确认

## 4b. 图库

- [ ] **空状态**：首次进入「图库」页，若没扫描过显示空状态 + 「扫描图库」按钮
- [ ] **扫描**：点「扫描图库」，底部进度条经过「扫描图库 X/Y」并完成；网格填满缩略图（缩略图懒加载，滚动时按需出现）
- [ ] **筛选栏**：左侧按作者/文件夹/标签/格式/方向/分辨率/年份分区列出选项 + 计数；多选同一维度为「或」，跨维度为「与」；勾选后网格收窄、其他维度计数随之更新
- [ ] **搜索**：顶部搜索框按文件名/路径/PID/作者过滤
- [ ] **折叠筛选栏**：工具栏左侧按钮可隐藏/显示筛选栏
- [ ] **详情**：点缩略图弹出详情，显示 PID、来源、作者、尺寸、格式、大小、修改时间、路径；上一张/下一张可在筛选结果内切换
- [ ] **图片标签**：在详情里给某张图加/删标签；该标签立即出现在左侧「标签」筛选里，且重新扫描后仍保留
- [ ] **打开来源 / 打开文件夹**：详情里的两个按钮分别打开 Pixiv 作品页与资源管理器
- [ ] **重新扫描**：再次扫描后，未改动的图走 reused、不丢图片标签
- [ ] **后台增量索引**：索引缺失、扫描目录变化或超过 6 小时时，启动后在底部独立显示索引进度；切换页面和艺术家操作仍可正常进行
- [ ] **索引状态**：工具栏显示索引“最新 / 需要更新”状态；后台更新完成后自动变为最新
- [ ] **图库体检**：点 Doctor，检查数据库、失效保存路径、路径重叠、浏览器用户目录、隔离目录与索引状态；诊断过程不修改任何文件
- [ ] **诊断路径**：Doctor 结果里的路径可双击在资源管理器中打开
- [ ] **图片多选**：点缩略图左上角选择按钮不会打开详情；“选择当前结果”只选择当前筛选结果，再点可取消这些结果
- [ ] **收藏 / 评分 / 状态**：详情中可收藏、设为 0–5 星并切换“参考价值高 / 已用过 / 待整理”；重新扫描后仍保留
- [ ] **批量编辑**：多选后可分别选择“不修改 / 标记 / 清除”，并批量添加或移除以逗号/换行分隔的标签
- [ ] **Pixiv 标签转本地**：已有 Pixiv 标签时，可在详情或批量编辑中复制到本地标签；不会重复添加
- [ ] **素材筛选**：收藏、评分和整理状态出现在左侧筛选栏，计数和组合筛选正确
- [ ] **导出列表**：有选中项时只导出所选；无选中项时导出当前筛选结果；CSV 用 Excel 打开中文/日文不乱码

## 5. Settings 5 个子分类

切换左侧导航,5 个 section 都应独立显示:

### General
- [ ] **语言**:中/英切换,顶部 tab、按钮、modal 文案应实时切换
- [ ] **Python 命令**:python / py 切换,保存后下一次任务用新命令运行
- [ ] **项目根**:浏览选择文件夹,保存后所有 `.pixiv-pbd-manager/` 路径基于这里
- [ ] **数据库**:浏览另存为 .json,保存后切到 Artists 列表应为空(新库)
- [ ] **显示进度条百分比**:取消勾选后,进度条不再显示 `N%`(但速度还在)
- [ ] **版本信息**:显示当前版本与构建 commit;开发构建没有 commit 时显示占位而不报错
- [ ] **检查更新**:手动检查 GitHub 最新 Release;正确显示“已是最新”或新版本号,并可打开 Release / Changelog / 构建产物页面

### Folders
- [ ] **下载根** / **排除根** textarea + 加文件夹按钮
- [ ] **R-18 分文件夹**:勾选后下载 R-18 作品应保存到 `[R-18&R-18G]/` 子目录(需要实跑下载验证)

### Scan
- [ ] **在线解析 / 模糊搜索 / SSL fallback / 扫描本地子文件夹** 4 个复选框可独立切换
- [ ] **解析作品上限 / 模糊分数下限 / 检查更新页数** 3 个数字输入
- [ ] **检查更新页数 = 0**:`updates.check` 拉**所有**作品;`> 0`:只拉最新 N 页

### Browser
- [ ] **浏览器路径**:浏览选 .exe 文件(file 类型)
- [ ] **用户数据目录**:浏览选目录;**如果该目录在 download_roots 下,应弹警告并拒绝**(`unsafeUserDataDir`)
- [ ] **延迟 / 限制**:数字输入

### Cookie
- [ ] **同意复选框未勾时**:cookie 输入框 disabled
- [ ] **首次勾选**:弹 DisclaimerModal(声明 cookie 风险),有"取消 / 接受"两按钮
  - 取消:同意复选框保持未勾
  - 接受:勾上,cookie 输入框启用
- [ ] **取消同意(勾→未勾)**:立即调 `cookie.revoke`,后端清空已存 cookie;日志出现 `cookieRevoked`
- [ ] **查看声明**:右侧"查看声明"按钮可随时再看 disclaimer(只读模式,只有"关闭"按钮)
- [ ] **显示/隐藏 cookie**:右侧按钮在 password ↔ text 切换 input type

### 自动保存
- [ ] **自动保存**:修改任意设置后自动写入,不需要点击保存按钮;快速连续输入只在停顿后保存一次
- [ ] **关闭再启动**:所有 settings 应保留

## 7. 进度条 & 任务控制(footer)

### 进度条布局
- [ ] **长任务运行中**:footer 出现 1-2 行进度条(主任务 + 当前文件)
- [ ] **进度条长度固定**:不随左侧任务名长短抖动
- [ ] **百分比 + 速度**:右侧 meter 区显示 `42% · 12.5 MB/s`,**紧贴进度条**,不随左侧文字漂移
- [ ] **任务名过长**:超出宽度用 ellipsis `...`,hover 显示完整 tooltip

### 暂停 / 恢复
- [ ] 任务运行中点 **Pause**:进度条停在当前数值,按钮变成 **Resume**,状态栏 `任务已暂停`
- [ ] 点 Resume:继续推进,按钮回到 Pause
- [ ] 暂停期间已下载到一半的文件**不被中断**(实测:下载大文件时,暂停按下后那个文件应该正常完成,不会中途断)

### 取消
- [ ] **Cancel**(红):立即终止;footer 出现 `任务已取消`;Python 子进程在任务管理器中消失(不留 zombie)
- [ ] 取消后再点同类任务能正常启动新一轮(refs 已清)
